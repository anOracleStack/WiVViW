## 0. HOW TO USE THIS (the fish-finder loop)
1. Take the product's **meaning** → generate 15–20 candidate words.
2. For each word, look at its **ending sound** → pull the fused-TLD options from §2.
3. **Bulk-check** the whole batch live (Spaceship Beast Mode or the Domainr/RDAP tool in §5) BEFORE getting attached.
4. Keep only **available** survivors → check **handles** (§4) → **score & rank** (§6).
5. #1 down. Decide.

Generation is cheap & wide. Verification gates everything. Scoring happens only on survivors.

## 3. RESTRICTED — verify eligibility BEFORE you fall in love
These have residency/nexus/trademark requirements that became a problem with viv.id (.id) earlier.

| TLD | Requirement |
| :-- | :-- |
| **.us** | US citizen/resident/org or genuine US presence (nexus) |
| **.id** | Indonesian local presence OR matching trademark — else paid trustee (registrar becomes legal holder, recurring fee ≈ doubles cost) |
| **.it / .eu** | EU/EEA presence required |
| **.ca / .au** | Canadian / Australian presence required |
| **.in** | Mostly open now, but confirm current policy |
| **.ly** | Routes through Libyan registry; content rules + stability risk — treat as caution, not core |
| **.de** | Easier now, but a German admin-c historically helped |

## 4. HANDLE CONSISTENCY (check in the same pass)
A name isn't "clear" until the handles are too. For a dev-talent portfolio, **GitHub is first-class.**

**Check each candidate on:** X · Instagram · TikTok · YouTube · LinkedIn (company vanity URL) · **GitHub (org)** · Discord · Reddit.
**Fallback convention (pick ONE, apply everywhere):** `get___` · `try___` · `join___` · `___hq` · `___app` · `hey___`.
**Tools:** Namechk / KnowEm for the multi-platform first pass; confirm survivors on each platform directly.
**Edge:** fused-TLD & mirror marks lean on the *domain* as the brand, so handle drift hurts less — but consistency still signals "on point."

## 5. DATA SOURCES & THE TOOL (your "we need to make it" — verified buildable)

**Canonical list of every TLD that exists:** the **IANA Root Zone Database** (~1,400+ delegated TLDs). This is the master from which §2 is curated.

**Live availability — three real paths (confirmed this session):**
| Source | What it gives | Access | Best for |
| :-- | :-- | :-- | :-- |
| **Spaceship API** | Availability + register/transfer/renew on YOUR platform | `spaceship.dev/api/v1/domains/{domain}/available`, key+secret, 30 checks/30s | Building on the registrar you already use |
| **Domainr / Fastly Domain Research API** | ICANN-accredited cross-TLD status, no false positives; "find available TLDs for a word" | RapidAPI (free/non-commercial) or direct (commercial) | The universal fish-finder |
| **RDAP** (replacing WHOIS) | Registration status, no key | Public protocol | Zero-cost fallback |
| **`domain-mcp`** (open-source) | Bulk availability (≤20), WHOIS, suggestions across 25+ TLDs — **connects to Claude, no API key** | MCP server | Agent-driven checks today |

**The build (this is oibid.io's core feature):**
```
INPUT: a meaning/concept (or a seed word)
  → 1. GENERATE candidate words (LLM, on-brand: coined/mythic/Latin + mirror-mark logic)
  → 2. FAN OUT each word across the §2 fused-TLD bank + workhorses
  → 3. CHECK availability (Spaceship API or Domainr/RDAP) — batch
  → 4. CHECK handles (Namechk-style) across platforms
  → 5. SCORE (memorability · distinctiveness/SEO · TM-risk · fusion-quality · handle-consistency)
  → 6. RANK → output only AVAILABLE, on-brand, handle-consistent names
OUTPUT: ranked shortlist where every entry is real & ownable
```
Raw availability is a commodity (solved by the APIs above). **Your moat is steps 1, 2, 5** — the branding intelligence. That layer doesn't exist as a polished product anywhere. That's the gap you spotted. That's oibid.io.

## 6. SCORING RUBRIC (survivors only)
Score each /5; rank #1 down. Domain/handle availability is now **confirmed**, not assumed (that's the point of the loop).

| Dimension | Test |
| :-- | :-- |
| Memorability | Recalled after one exposure? |
| Pronounceability | One obvious spoken form? |
| Spellability | Spelled right from hearing? |
| Distinctiveness / SEO | Ownable SERP, no big-brand collision? |
| Fusion quality | Does the TLD complete the word *cleanly & meaningfully*? |
| Meaning fit | Captures the product's concept? |
| Handle consistency | Same string (or one convention) across platforms? |
| TM headroom | Quick search shows no funded company / Class 35-42 conflict? |
