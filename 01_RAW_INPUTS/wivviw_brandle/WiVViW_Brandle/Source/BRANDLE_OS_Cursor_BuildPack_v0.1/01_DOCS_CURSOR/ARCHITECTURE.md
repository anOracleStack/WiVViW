# BRANDLE OS — Architecture (Cursor)

## Layer 0: Shared OS Services (always present)
- Brand Genome (canonical project object)
- Confidence System (Estimate/Likely/Verified)
- Receipts System (inputs → transforms → outputs)
- Evidence Vault (proof items; claim upgrades)
- V4ULT Timeline (versioning for runs + exports)
- Export Chamber (TXT/PDF/JSON bundles; watermark rules)
- Governance (NDA gating; access logs; audit stamps)

## Layer 1: Engines (modules)
- dRANb: naming generation + scoring + testing labs
- brandL: identity consolidation + ladders + compromise cost
- 4IELD: naming/market field mapping + drift
- 4TRESS: protection posture (non-legal; readiness + flags)

## Layer 2: Presentation Engine (cinematic skin)
- Backdrop Engine (seeded selection)
- Texture system (grain/bloom)
- Motion grammar (calm, procedural)

## MVP Backend strategy (optional)
- Start with localStorage
- Move to:
  - Postgres (projects, runs, evidence, exports)
  - Object storage (evidence files, export bundles)
  - Queue (drift checks, background computations)
